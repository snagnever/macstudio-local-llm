# 2026-05-20 — Phase 2 Gemma 4 full-suite benchmark run

## Context

[`docs/testing-plan.md`](../testing-plan.md) Phase 1 closed 2026-05-19 with
the three Qwen daily-driver candidates fully characterized. Phase 2 is the
"already on disk, zero download cost" set — the four Gemma 4 variants. The
decisions Phase 2 is meant to settle:

- Does **`gemma-4-26b-a4b@4bit`** beat the 35B-A3B in the "fast generalist"
  slot at less than half the on-disk size?

  quant-robust at this scale?
- Does **`gemma-4-31b-dense`** (8-bit) justify the dense tax vs. the MoE-26B?
- Is **`gemma-4-e4b`** (4B/8-bit) viable for the FIM / quick-call slot?

All 4 models verified live in LM Studio (`GET http://192.168.68.124:1234/v1/models`,
2026-05-20). No prior Gemma 4 result folders exist in either harness. Scope:
**full suite, all 4 models, in Phase 2 numbering order
(`@4bit → @6bit → 31B-dense → E4B`)**.

## Per-model run order

Cheap signal first; abort and reprioritize if a cheap step disqualifies the
model.

1. **Effective-throughput sweep** (~15–25 min) — `local-llm-bench/bench.py`,
   runs all 4 scenarios (ops-agent, doc-summary, prefill-test, creative-writing).
   Doubles as the speed probe.
2. **Tool-calling** (~30 min) — `tool_call_bench.py --suite both` (jdhodges + Veerman).
3. **HumanEval** (~30–60 min) — `bench2.py humaneval --examples 100`.
4. **LiveCodeBench** (~30–90 min) — `bench2.py livecodebench --examples 50 --lcb-version release_v6`.
5. **MMLU** (~30–60 min) — `bench2.py mmlu --examples 100`.
6. **MATH** (~2–8 h) — `bench2.py math --examples 100`.
7. **DROP** (~1–2 h) — `bench2.py drop --examples 100`.
8. **GPQA** (~2–10 h) — `bench2.py gpqa --examples 100 --max-tokens 65536` ← **mandatory flag** per
   [testing-plan Truncation finding](../testing-plan.md). Gemma 4 may or may not
   spiral the way Qwen3.6 thinking models did, but apply the rule preemptively
   so a 2026-05-19-style trap doesn't repeat.

Run **model-major** (entire suite on Gemma #1 before starting #2). Interrupted
runs then leave at least one model fully characterized.

Tool use & agents Terminal-Bench and tool-calling suites

Terminal-Bench 2.0 — local (n=89)

Multi-turn shell-agent benchmark, measured on-rig 2026-05-24 → 2026-05-29 via Harbor + LiteLLM → LM Studio (Docker linux/amd64), terminus-2 agent, concurrency=1, vanilla T-Bench 2.0 task set with --agent-timeout-multiplier 0.5 (bounds the 14 tasks declaring >60-min agent budgets — scores are a defensible floor; full-budget would land ≤5 pp higher). All 7 candidates complete: qwen/qwen3-coder-next 32.6 % (29/89, 16.8 h), qwen3.6-27b 31.5 % (28/89, 18.9 h), qwen3.6-35b-a3b@6bit 28.1 % (25/89, 15.6 h), gemma-4-31b dense 22.5 % (20/89, 17.4 h), gemma-4-26b-a4b@6bit 21.3 % (19/89, 14.4 h), @4bit 20.2 % (18/89, 16.0 h), gemma-4-e4b 4.5 % (4/89, 6.4 h). Total wall-clock ~105 h. Headline: T-Bench cleanly inverts LCB at the top — the three Qwen models cluster on top (trained with explicit agentic-loop targets); Gemma — which topped LCB at 80 % — drops to mid-pack. Canonical summaries: tools/local-llm-bench-m4-32gb/benchmarks/runs/tbench_*_summary.json; raw run dirs at bench/terminal-bench/logs/tbench-runs/<model>/result.json.
